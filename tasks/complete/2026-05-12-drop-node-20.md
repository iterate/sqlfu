# Drop Node 20 support

## Status summary

Done. Internal sqlite paths now require `node:sqlite`, root and package metadata
declare Node 22+, and the obsolete `better-sqlite3` build-script rationale is
gone. `better-sqlite3` stays as a sqlfu dev dependency because real adapter,
recipe, error, and authority tests still open that driver directly.

## Assumptions for this pass

- `node:sqlite` is the default internal sqlite host now that the minimum runtime
  is Node 22.
- The public `better-sqlite3` adapter tests should stay if they intentionally
  prove the adapter still works for users.
- `better-sqlite3` should only leave `packages/sqlfu/package.json` if every
  remaining reference is type-only, generated output, or otherwise not required
  by real tests.
- Root `README.md` should be regenerated through the existing docs flow if the
  source README changes, rather than edited as the source of truth.

CI already runs on Node 22 (bumped in the `config-db-pluggable` PR because
Node 20 can't strip type-only imports from the vendored sql-formatter).
The runtime still has Node-20 fallbacks that can come out.

- [x] `openMainDevDatabase` in `packages/sqlfu/src/typegen/index.ts` —
  drop the `better-sqlite3` fallback, leave only the `node:sqlite`
  path. Delete the "Node 22" explanation comment. _implemented in
  `packages/sqlfu/src/typegen/index.ts`; the Node path imports `node:sqlite`
  directly and no longer catches `ERR_UNKNOWN_BUILTIN_MODULE`_
- [x] `scripts/generate-internal-queries.ts` — stop importing
  `better-sqlite3`; use `node:sqlite` (or just go through the node host
  like typegen now does). Delete the Node-20 comment. _the script is absent on
  `origin/main`; the remaining stale reference was the `pnpm-workspace.yaml`
  build-script comment, which now points at the real adapter tests_
- [x] Root + package `engines.node` — set to `>=22`. _added to root
  `package.json` and `packages/sqlfu/package.json`_
- [x] `packages/sqlfu/package.json` devDependencies — remove
  `better-sqlite3` if no tests still need it (grep: `errors.test.ts`,
  `core-sqlite.test.ts`, `generate-authority.test.ts`,
  `recipes/id-helpers.test.ts`, `test/better-sqlite3.d.ts` — these
  assert the adapter itself works, keep them). _kept because real tests still
  import and open `better-sqlite3`; `pnpm-workspace.yaml` still allows its native
  build for those tests_
- [x] README + docs — any "supports Node 20+" phrasing becomes
  "Node 22+". _no matching README/docs phrases were present on this base; `rg`
  found only the now-removed code comments_
