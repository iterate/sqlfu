## medium: Root test script skips the @sqlfu/pg test suite

id: fnd_sig-feat-config-7528cb5b98-eb46c_f96db235a2
category: test-gap
confidence: high
triage: test-gap
status: open
feature: Project config package.json (feat_config_7528cb5b98)

evidence:
- package.json:28 (scripts.test)
- package.json:26-29 (scripts)
- packages/pg/test/diff-schema.test.ts:17-48 (pgDialect.diffSchema tests)
- packages/pg/test/materialize-schema.test.ts:17-48 (pgDialect.materializeSchemaSql tests)
- packages/pg/test/migration-lock.test.ts:23-68 (withMigrationLock concurrency test)

The workspace build and typecheck scripts treat @sqlfu/pg as a first-class package, and the feature metadata lists pg tests under the expected `pnpm test` command. However, the root `pnpm test` script only runs `sqlfu` and `@sqlfu/ui`, so pg regression coverage is silently omitted from the standard quality command. That means schema diff, materialization, fixture, and migration-lock regressions can pass CI or local checks when developers run the advertised root test command.

recommendation:
Include `pnpm --filter @sqlfu/pg test` in the root `scripts.test`, preferably between sqlfu and ui to mirror the package ordering used by build/typecheck.

test analysis:
The skipped tests live under `packages/pg/test`, but the root script does not invoke that package's test command, so they cannot detect their own omission from the workspace-level command.

suggested regression test:
Add or update a package-script validation test, if the repo has one, that asserts every package included in root build/typecheck and exposing a test script is also included in root `pnpm test`.

minimum fix scope:
Edit `package.json` root `scripts.test` to include `pnpm --filter @sqlfu/pg test`.

repro:
Run `pnpm test` from the workspace root and observe that it invokes only `pnpm --filter sqlfu test` and `pnpm --filter @sqlfu/ui test`, never `pnpm --filter @sqlfu/pg test`.
